Data-quality checks, right on the record page

Record Health Check is a metadata-driven framework for running data-quality checks against a Salesforce record, right on its record page. It is advisory and read-only, so it never blocks a save and never writes a field.

Install

Up and running in minutes

The fastest path is the Deploy to Salesforce button — click it, log in to a sandbox, and click Deploy. No command line, no downloads. Prefer the CLI? Deploy the source directly.

deploy with the Salesforce CLI 
$ git clone https://github.com/gkolan/recordHealthCheck.git
$ cd recordHealthCheck
$ sf project deploy start --source-dir force-app
$ sf org assign permset --name Record_Health_Check_User
Then add the card to a record page
1
On a Lightning record page, drag on the recordHealthCheck component.
2
Set its Check Set Developer Name to a Check Set's Developer Name, for example Account_Data_Quality. It must match exactly — a mismatch is the single most common misconfiguration.
3
Save and activate, then open a record to see the card evaluate it.

What it does

Four ways to check a record

Validation rules enforce at save time, one record and one field at a time. Record Health Check complements them by running at read time — so it can evaluate across related records, compare aggregates, and surface data that predates your rules.

Formula checks
Evaluate a formula over fields on the current record (requires org API v63.0+).
Query checks
Run a SOQL query and compare the result to a value or a list.
Compare two queries
Run two queries and compare their results to each other.
Apex checks
Call a custom Apex class for logic beyond formulas and SOQL.
Applicability & dependencies
Decide per record whether a Rule runs, and let one Rule wait for another to pass first.
Automatic or on-demand
Evaluate the record on page load, or when you click Run.
